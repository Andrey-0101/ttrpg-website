# Architecture Decision Records

## Format

Each ADR contains:

- status;
- context;
- decision;
- consequences;
- alternatives;
- follow-up.

## Status values

```text
Proposed
Accepted
Superseded
Rejected
Deprecated
```

Accepted ADRs should not be silently rewritten when the decision changes. Create a new ADR and mark the old one superseded.
